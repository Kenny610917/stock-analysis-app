# Stock Analysis Skill：让 AI Agent 做更像投研员的股票分析

> 面向 Codex、Claude Code、OpenAI Agents 等 AI Agent 的专业个股投研 Skill。  
> 关键词：Stock Analysis、A 股投研、财务排雷、市值倒推、逆向估值、关键因子跟踪。

普通 AI 股票分析最常见的问题，是顺着公司故事往下写：行业空间很大、公司增长很快、估值似乎可以接受。  
这个 Skill 的目标正相反：**先把故事拆开，再问价格是否已经透支未来。**

它会强制 AI Agent 区分三件事：

1. 公司是否优秀。
2. 增长是否真实。
3. 当前价格是否已经透支。

> 风险提示：本项目仅用于公开资料研究、投研学习和 AI Agent 工作流演示，不构成任何投资建议。

## 为什么需要它

当你让通用 AI 分析股票时，它往往会犯这些错误：

- 只看利润增长，不看经营现金流、应收账款和存货。
- 只写行业空间，不倒推当前市值隐含了多高的未来业绩。
- 只列风险提示，不告诉你什么事实出现会推翻结论。
- 只输出一次性报告，不给后续跟踪的关键因子和红黄绿灯阈值。

`stock-analysis-skill` 把这些坑变成固定流程，让 AI Agent 像投研员一样做逆向验证。

## 核心能力

| 能力 | 解决的问题 |
|---|---|
| 五步法分析 | 好公司、好未来、好价格、好买卖、风险提示 |
| 财务排雷 | 检查净现比、回款率、应收、存货、大存大贷 |
| 市值倒推 | 反推当前市值要求未来做到多少收入和利润 |
| 关键因子跟踪 | 输出红黄绿灯阈值和信息源 |
| 反证清单 | 明确哪些事实出现会推翻当前结论 |
| 数据源分级 | 区分公告、AkShare、研报、新闻、社媒的可信度 |

## 快速开始

### 方式一：复制到 Codex / Agents 技能目录

把本仓库复制到你的项目技能目录：

```text
你的项目/
└── .agent/
    └── skills/
        └── stock-analysis/
            ├── SKILL.md
            ├── references/
            ├── templates/
            └── examples/
```

然后在对话中使用：

```text
使用 stock-analysis 分析源杰科技 688498，重点看估值是否透支。
```

### 方式二：手动读取 Skill

如果你的 Agent 暂不支持技能目录，也可以直接让它读取 `SKILL.md`：

```text
请读取 ./stock-analysis-skill/SKILL.md，并按其中流程分析某只股票。
```

## 示例：源杰科技逆向分析片段

> 完整示例见 [`examples/yuanjie-reverse-analysis.md`](examples/yuanjie-reverse-analysis.md)。

| 目标 PE | 市场隐含全年净利润 | Q2 至少需要净利润 | 按 50.51% 净利率倒推 Q2 营收 |
|---:|---:|---:|---:|
| 200 倍 | 7.09 亿元 | 1.75 亿元 | 3.47 亿元 |
| 150 倍 | 9.46 亿元 | 2.94 亿元 | 5.82 亿元 |
| 120 倍 | 11.82 亿元 | 4.12 亿元 | 8.16 亿元 |
| 100 倍 | 14.19 亿元 | 5.30 亿元 | 10.49 亿元 |

这个表的价值在于：它不只说“公司增长快”，而是继续追问：**当前价格到底要求下一季度交出什么成绩单？**

## 数据源原则

本 Skill 鼓励使用 AkShare、东方财富、新浪财经等结构化数据做初筛，但核心结论必须优先以公司公告、年报、季报、交易所文件为准。

数据可信度默认排序：

1. 公司公告、年报、季报、交易所文件。
2. 投资者关系活动记录、业绩说明会、交易所互动平台。
3. AkShare、东方财富、新浪财经、同花顺等结构化数据。
4. 券商研报、行业报告、协会数据、统计局数据。
5. 新闻媒体、社交媒体、市场传闻。
